## Data Flow: Real-Time Example

```
SCENARIO: Blog daily loop completes

1. [Local] blog-daily-master loop finishes
   └─ Writes to ~/.agents/.harness/progress/session-log.md

2. [Python] collect-live-data.py runs (every 5 min)
   └─ Reads session-log.md
   └─ Updates live-data.json

3. [Server] api-endpoints.js refreshes cache (every 5 sec)
   └─ Reads live-data.json
   └─ Updates dataCache.loops

4. [WebSocket] Server broadcasts to ALL connected clients
   └─ wss.clients.forEach(client => client.send(update))

5. [Frontend] websocket-client.js receives update
   └─ Emits 'data-update' event
   └─ Updates doorDashboardState

6. [Dashboard] renderLoopDashboard() detects change
   └─ Re-renders loop table with new data
   └─ Shows latest status, uptime, success rate

7. [Browser] User sees update INSTANTLY
   └─ No manual refresh needed
   └─ <500ms latency (over WebSocket)
```

## Architecture Layers

### Layer 1: Frontend (Browser)
- **tracker_live.html** — Main dashboard HTML
- **websocket-client.js** — WebSocket connection manager
  - Auto-reconnect with exponential backoff
  - Event listener system
  - Cache local data
- **dashboards-the-door-v2.js** — Rendering logic
  - Async + real-time updates
  - State management (doorDashboardState)
  - Alert notifications

**Key Features:**
- Zero polling (no wasteful HTTP requests)
- Real-time updates via WebSocket
- Graceful degradation if server unavailable
- Auto-refresh UI when data changes

### Layer 2: API Server (Node.js)
- **api-endpoints.js** — Express + WebSocket server
  - Port 3456 (configurable via $PORT)
  - Data cache (in-memory)
  - Refresh cycle (5 seconds)
  - Alert detection + broadcasting
  - n8n webhook receiver

**Endpoints:**
```
REST API:
GET  /api/health                     ← Health check
GET  /api/loops/{active|all|:id}     ← Loop status
GET  /api/context/tokens             ← Token usage
POST /api/context/track              ← Track tokens
GET  /api/projects/metrics           ← Project KPIs
GET  /api/roadmap                    ← Q2/Q3/Q4 phases
GET  /api/alerts                     ← Alert history
POST /api/alerts                     ← Create alert
POST /api/webhook/n8n                ← Receive from n8n

WebSocket:
ws://localhost:3456/                 ← Real-time data
```

**Alert Logic:**
```javascript
// Every 60 seconds:
if (loop.failures / loop.attempts > 0.1) {
  // Send alert (severity: warning)
  broadcast({ type: 'alert', data: { ... } });
  sendToN8n({ type: 'alert', ... }); // Email/Telegram
}

if (context.percentUsed > 90) {
  // Send alert (severity: critical if > 95%)
  broadcast({ type: 'alert', data: { ... } });
}
```

### Layer 3: Data Collection (Python)
- **collect-live-data.py** — Data aggregator
  - Runs every 5 minutes (via Task Scheduler)
  - Reads from:
    - ~/.agents/.harness/progress/*.md
    - ~/.claude/memoria-sesiones/clientes-activos.json
    - Calculated values
  - Outputs: live-data.json

**Task Scheduler Setup:**
```powershell
$trigger = New-ScheduledTaskTrigger -RepetitionInterval (New-TimeSpan -Minutes 5)
$action = New-ScheduledTaskAction -Execute "python" `
  -Argument "C:\Users\inbou\victor-ia-tracker\collect-live-data.py"
Register-ScheduledTask -TaskName "Victor-IA-Collector" `
  -Trigger $trigger -Action $action
```

### Layer 4: Integration (n8n)
- **Webhook Receiver:** `POST /api/webhook/n8n`
- **Webhook Sender:** `sendToN8n()` on critical alerts
- **Workflow:** Send notifications
  - 🔴 Critical alerts → Email + Telegram
  - 🟡 Warnings → Email only
  - ✅ Success updates → Log only

## Troubleshooting

### WebSocket Connection Failed
**Cause:** API server not running  
**Fix:**
```bash
node api-endpoints.js
# Or check if port 3456 is in use:
netstat -an | grep 3456
```

### Data Not Updating
**Cause:** collect-live-data.py not running  
**Fix:**
```bash
python collect-live-data.py  # Manual run
# Check output: live-data.json
# Or setup Task Scheduler
```

### "Cannot GET /api/loops/active"
**Cause:** API server offline or wrong port  
**Fix:**
```bash
curl http://localhost:3456/api/health
# If fails: start server
# If 404: check route in api-endpoints.js
```

### WebSocket Reconnecting Continuously
**Cause:** Server rejecting connections  
**Fix:**
- Check API server console for errors
- Verify CORS headers in api-endpoints.js
- Check firewall (port 3456)
